# Repository instructions

## Upstream attribution

When implementation work references, studies, adapts, or derives ideas or code from an upstream repository or other
external source:

1. Record the source in the repository's root [`ATTRIBUTION.md`](ATTRIBUTION.md).
2. Include the source name and canonical URL, the revision reviewed when known, and its declared license when known.
3. State concretely which ideas, patterns, behavior, or code were used and which major pieces were intentionally not
   adopted.
4. Distinguish conceptual inspiration from copied or modified code. Preserve all license notices required for copied or
   modified material.
5. Update the attribution in the same change that introduces or materially changes the derived implementation.

Do not rely only on commit messages, pull-request text, or chat history for upstream attribution.
